## 常见场景

* 网站突然down 了，开发人员不知道
* 网站证书突然过期了，开发人员不知道
* 服务器资源占用突然增多，不知道是在攻击还是怎么回事
  * 内存使用增大
  * CPU 占用增大
  * 网络流量增大
* api 服务停止工作了，开发不知道。

## 解决之道

* 对url进行监控（Status code、性能指标、证书到期时间），设置阈值报警，通知到钉钉。
* 对服务器主机进行监控（CPU、内存、负载、磁盘 IO、网络流量），设置阈值报警，通知到钉钉。
  * 
* 对 docker 监控（Docker 中一版部署着特定服务），监控指标，每个 Docker 占用的主机资源，包含cpu、内存、流量、磁盘 IO 等信息。
